# Global Environment & The Global Object

## 1. Global Environment

**Definición:**  
Entorno léxico más externo donde inicia la ejecución de JavaScript.

**Componentes:**

### 1. Environment Record Global
Registro interno donde se guardan todas las variables globales (fuera de funciones o bloques).

`js
var x = 10;  // se guarda en el Global Environment
let y = 20;  // también se guarda
`

### 2. Outer Environment Reference
Valor: 
ull (no tiene padre, está en la raíz).

### 3. Vinculación al Global Object
El Global Environment se compone de:
- **Global Lexical Environment** (let/const)
- **Global Variable Environment** (var y funciones)

Conectados al **global object**.

---

## 2. Global Object

**Definición:**  
Objeto especial creado automáticamente por el motor de JS al iniciar la ejecución.

**Variantes según el entorno:**
- **Navegador:** window
- **Node.js:** global
- **Web Workers:** self

**Contenido:**

### Funciones globales
parseInt(), isNaN(), etc.

### Objetos nativos
Math, JSON, Date, etc.

### APIs del host
En navegador: document, localStorage, etch, etc.

---

## 3. Relación entre Global Environment y Global Object

### El Global Environment usa al Global Object como base

El Global Object provee sus propiedades como variables globales.

**Ejemplo:**
`js
console.log(Math.PI);  // Math existe porque está en el global object
`

### Variables ar se agregan al Global Object

`js
var a = 10;
console.log(window.a);  // 10 en navegador
`

✓ Las ar se convierten en propiedades del global object.

**Pero:**
`js
let b = 20;
console.log(window.b);  // undefined
`

✗ let y const NO se agregan al global object; se guardan en el global lexical environment.

---

## Esquema Resumido

`
Global Environment
 ├─ Global Lexical Environment (let/const)
 ├─ Global Variable Environment (var)
 └─ vinculado al → Global Object (window / global)
`
