# Single-Threaded Model y Synchronous Execution

## 1. Single-Threaded Model

**Definición:**  
JavaScript solo tiene un hilo de ejecución principal. Solo puede ejecutar una tarea a la vez.

**Características:**
- Solo existe un Call Stack
- No puede ejecutar dos líneas de código simultáneamente
- No puede correr funciones en paralelo (dentro del mismo hilo)
- Si una tarea tarda mucho, bloquea todo el programa
- No hay carreras de datos internas en el hilo principal

**Ejemplo:**
```js
console.log("A");
console.log("B");
console.log("C");
```

Salida siempre:
```
A
B
C
```

---

## 2. Synchronous Execution

**Definición:**  
El motor ejecuta cada línea en orden y no avanza a la siguiente hasta que la actual termina.

**Características:**
- Modelo bloqueante
- La instrucción A debe terminar antes de que empiece B
- Ejecución paso por paso

**Ejemplo:**
```js
function tareaLenta() {
  for (let i = 0; i < 2_000_000_000; i++) {}
}

console.log("Inicio");
tareaLenta(); // bloquea todo
console.log("Fin"); // solo aparece cuando termina la anterior
```

---

## Relación entre ambos conceptos

**Single-Threaded** → solo hay un camino de ejecución  
**Synchronous** → ese camino se recorre paso por paso

**Significado conjunto:**  
JavaScript ejecuta el código en un único hilo y lo hace de manera bloqueante: una instrucción tras otra.

---

## Código asincrónico en JavaScript

**Pregunta:** ¿Cómo corre código asincrónico (timers, fetch, promesas)?

**Respuesta:**  
El navegador y Node.js agregan APIs externas (Timers, Web APIs, Libuv). Estas tareas se ejecutan fuera del hilo principal y regresan al Event Loop para ser procesadas.

**Importante:**  
El motor JavaScript en sí sigue siendo single-threaded.

---

## Resumen

| Concepto | Definición | Consecuencia |
|----------|-----------|--------------|
| **Single-Threaded Model** | Un solo hilo de ejecución | No ejecuta código en paralelo |
| **Synchronous Execution** | Cada instrucción debe terminar antes de avanzar | Ejecución bloqueante |
