# Syntax Parsers, Lexical Environments y Execution Contexts

## 1. Syntax Parser

**Definición:**  
Herramienta del motor de JavaScript que lee el código fuente y verifica si cumple con las reglas gramaticales del lenguaje.

**Funciones principales:**
- Analiza la estructura del código
- Construye el Abstract Syntax Tree (AST)
- Detecta errores de sintaxis antes de la ejecución
- No evalúa el significado, solo valida la forma correcta

---

## 2. Lexical Environment

**Definición:**  
Estructura interna creada cada vez que se genera un bloque {}, una función o un programa completo.

**Componentes:**
- Variables declaradas en ese entorno específico
- Referencia al entorno padre (outer environment)
- Reglas para la resolución de nombres/identificadores

**Característica clave:**  
El significado de un identificador depende de dónde fue **definido** (lexical scope), no de dónde se ejecuta.

---

## 3. Execution Context

**Definición:**  
Entorno donde se ejecuta código JavaScript. Se crea cada vez que JS evalúa código.

**Fases:**

### Fase de Creación:
1. Se crea el Global Object
2. Se establece el binding de `this`
3. Se crea la referencia al Outer Environment
4. Se configura el espacio en memoria (Hoisting):
   - Variable Environment: variables `var`
   - Lexical Environment: variables `let`/`const`
   - Function Arguments
5. Las variables `var` se inicializan con `undefined`

### Fase de Ejecución:
- El código se ejecuta línea por línea
- Se asignan los valores a las variables

**Organización:**  
Los Execution Contexts se apilan en el **Call Stack** (pila de llamadas).
