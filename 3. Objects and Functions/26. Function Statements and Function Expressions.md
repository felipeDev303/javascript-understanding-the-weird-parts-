# Function Statements and Function Expressions en JavaScript

En JavaScript existen dos maneras clásicas de crear funciones: **Function Statements** (también llamados _Function Declarations_) y **Function Expressions**. Ambas crean funciones, pero se comportan de manera distinta en términos de **cuándo** están disponibles y **cómo** se asignan en el código.

---

## 1. Function Statement (Function Declaration)

Un **Function Statement** declara una función con un nombre y la coloca en el entorno de ejecución durante la fase de _hoisting_.
Esto significa que la función está disponible **antes** de que el código se ejecute.

### Sintaxis:

```js
function saludar() {
  console.log("Hola");
}
```

### Características:

- Se cargan completamente durante el hoisting.
- Se pueden invocar antes de su declaración en el código.
- Siempre generan una función con nombre.

### Ejemplo:

```js
saludar(); // funciona

function saludar() {
  console.log("Hola");
}
```

---

## 2. Function Expression

Una **Function Expression** crea una función como parte de una expresión, usualmente asignada a una variable.
No es hoisted de la misma forma que un Function Statement; solo se hace hoisting del nombre de la variable, pero la función **no está disponible** antes de la asignación.

### Sintaxis básica:

```js
const saludar = function () {
  console.log("Hola");
};
```

### Variantes:

- Función anónima (más común)
- Función nombrada (menos usada)
- Arrow Function (forma moderna)

### Características:

- No se pueden usar antes de la línea donde se crean.
- Pueden ser anónimas o tener nombre.
- Más flexibles para usar en callbacks, IIFEs, funciones como valores, etc.

### Ejemplo:

```js
saludar(); // error: saludar is not a function

const saludar = function () {
  console.log("Hola");
};
```

---

## 3. Diferencias principales

| Aspecto                               | Function Statement                  | Function Expression                                 |
| ------------------------------------- | ----------------------------------- | --------------------------------------------------- |
| Hoisting                              | Sí (disponible antes de ejecutarse) | No (solo se eleva la variable, no la función)       |
| Uso antes de su definición            | Permitido                           | No permitido                                        |
| Nombre                                | Siempre nombrada                    | Puede ser anónima o nombrada                        |
| Flexibilidad                          | Menos flexible                      | Muy flexible (callbacks, asignaciones, estructuras) |
| Comportamiento en tiempo de ejecución | Se carga en fase de creación        | Se crea durante la fase de ejecución                |

---

## 4. Ejemplo comparativo

### Function Statement:

```js
caminar(); // OK

function caminar() {
  console.log("caminando...");
}
```

### Function Expression:

```js
correr(); // Error

const correr = function () {
  console.log("corriendo...");
};
```

---

## 5. Uso común de Function Expressions

### Funciones anónimas:

```js
setTimeout(function () {
  console.log("Hola");
}, 1000);
```

### Funciones nombradas:

```js
const f = function procesar() {
  console.log("procesando...");
};
```

### Arrow functions:

```js
const sumar = (a, b) => a + b;
```

---

## 6. Conclusión

- Un **Function Statement** define una función que se eleva completamente y se puede usar antes de su declaración en el código.
- Una **Function Expression** crea una función como valor dentro de una expresión; no está disponible antes de su asignación.
- Ambos métodos crean funciones válidas, pero las Function Expressions son más flexibles y ampliamente usadas en programación moderna, especialmente para callbacks, composición de funciones y patrones como módulos.
