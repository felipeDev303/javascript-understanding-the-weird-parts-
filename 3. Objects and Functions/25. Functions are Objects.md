# Functions Are Objects en JavaScript

En JavaScript, **las funciones son objetos**. Esto significa que, además de poder ejecutarse, también poseen propiedades, métodos y pueden manipularse igual que cualquier otro objeto del lenguaje.

A diferencia de otros lenguajes donde las funciones son entidades especiales o separadas, en JavaScript las funciones se crean a partir del tipo interno **`Function`**, que es un constructor de objetos. Por lo tanto, una función es simultáneamente:

1. **Un objeto** (con propiedades, métodos y un prototipo).
2. **Un callable** (algo que puede ser invocado con `()`).

---

## Características principales

### 1. Una función tiene propiedades como cualquier objeto

Ejemplo:

```js
function saludar() {
  console.log("Hola");
}

saludar.nombre = "función de saludo";
saludar.version = 1.0;

console.log(saludar.nombre); // "función de saludo"
console.log(saludar.version); // 1
```

Las funciones permiten añadir propiedades personalizadas de la misma forma que un objeto literal.

---

### 2. Las funciones tienen métodos internos (por ejemplo, `.call`, `.apply`, `.bind`)

Estos métodos existen porque **las funciones están basadas en el objeto `Function.prototype`**.

```js
function hablar(mensaje) {
  console.log(mensaje);
}

hablar.call(null, "Hola con .call");
hablar.apply(null, ["Hola con .apply"]);
const nuevaFuncion = hablar.bind(null, "Hola con .bind");
nuevaFuncion();
```

---

### 3. Las funciones tienen un prototipo propio que permite crear objetos

Todas las funciones en JavaScript poseen una propiedad `prototype`, usada como base para crear objetos mediante el patrón de constructor (antes de ES6).

```js
function Persona(nombre) {
  this.nombre = nombre;
}

Persona.prototype.saludar = function () {
  console.log("Hola, soy " + this.nombre);
};

const p = new Persona("Ana");
p.saludar();
```

---

### 4. Las funciones pueden ser pasadas como valores

Las funciones, al ser objetos, pueden asignarse a variables, pasarse a otras funciones o devolverse desde ellas.

```js
function operar(fn) {
  fn();
}

const mensaje = () => console.log("Ejecutado");

operar(mensaje);
```

---

### 5. Las funciones pueden almacenarse en estructuras de datos

También se pueden guardar en arrays, objetos, maps, etc.

```js
const acciones = {
  iniciar: () => console.log("iniciando"),
  detener: () => console.log("deteniendo"),
};

acciones.iniciar();
```

---

## Por qué es importante que las funciones sean objetos

- Permite **programación funcional** (callbacks, funciones de orden superior).
- Permite **construir objetos** antes de ES6 (patrón constructor).
- Fundamenta patrones como **Module Pattern** y **Revealing Module Pattern**.
- Permite manejar funciones como **datos**, lo que hace posible `setTimeout`, `.map`, `.filter`, promesas, middleware, etc.
- Hace posible usar funciones como **clases** antes de `class` en ES6.

---

## Resumen de puntos clave

- En JavaScript **una función es un objeto** creado por el constructor interno `Function`.
- Las funciones pueden tener **propiedades y métodos personalizados**.
- Las funciones incluyen métodos nativos como `.call`, `.apply` y `.bind`.
- Las funciones poseen una propiedad **`prototype`**, usada para crear objetos (antes de ES6).
- Las funciones pueden pasarse como argumentos, retornarse y almacenarse en estructuras, porque **son valores**.
- Este concepto es la base de la programación funcional, constructores, módulos y patrones avanzados del lenguaje.
