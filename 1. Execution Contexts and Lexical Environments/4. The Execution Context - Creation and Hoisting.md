# Hoisting

## Definición

Comportamiento del motor de JavaScript donde las declaraciones (no las asignaciones) se procesan antes de ejecutar el código.

**Proceso real:**  
El motor registra las declaraciones en la fase de creación del Execution Context, antes de ejecutar la primera línea del programa.

---

## Comportamiento según el tipo de declaración

### `var`

- Se declara al inicio del scope
- Valor inicial: `undefined`
- Accesible antes de la línea de declaración

```js
console.log(a); // undefined
var a = 5;
```

### Function Declarations

- Se eleva la declaración completa (incluye el cuerpo)
- Puede invocarse antes de su definición en el código

```js
saludar(); // "Hola"

function saludar() {
  console.log("Hola");
}
```

### `let` y `const`

- Sí se elevan, pero permanecen en la Temporal Dead Zone (TDZ)
- No accesibles hasta que la ejecución alcanza su línea de declaración
- Generan ReferenceError si se acceden antes

```js
console.log(b); // ReferenceError
let b = 10;
```

---

## Funcionamiento Interno

Durante la **fase de creación** del Execution Context:

### 1. Variable Environment (para `var`)
- Registra variables `var`
- Las inicializa con `undefined`

### 2. Lexical Environment (para `let`/`const`)
- Registra los identificadores
- No asigna valor inicial
- Crea la Temporal Dead Zone (TDZ)

### 3. Function Declarations
- Se almacenan completas con su cuerpo
- Disponibles inmediatamente

---

## Resumen

- Las **declaraciones** se elevan, las **asignaciones** no
- `var` → hoisting + inicialización a `undefined`
- `function` → hoisting completo (declaración + cuerpo)
- `let`/`const` → hoisting + Temporal Dead Zone
- Ocurre en la fase de creación, antes de ejecutar código
