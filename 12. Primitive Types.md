# Primitive Types

## Definición

JavaScript trabaja con dos categorías de valores:
- **Primitivos (Primitive Types)**
- **Objetos (Object Types)**

**Primitivos:**  
Valores inmutables, de tamaño fijo, almacenados directamente en memoria (stack). No tienen comportamiento complejo y no se modifican: se reemplazan.

---

## Los 7 Tipos Primitivos

1. **string**
2. **number**
3. **boolean**
4. **undefined**
5. **null**
6. **symbol**
7. **bigint**

---

## Descripción de cada tipo

### 1. `string`

Representan texto.

```js
let nombre = "Felipe";
```

- Siempre entre comillas
- Son inmutables

### 2. `number`

JavaScript tiene un solo tipo numérico, para enteros y decimales.

```js
let edad = 25;
let precio = 19.99;
```

Basado en IEEE 754, a veces verás:
```js
0.1 + 0.2;  // 0.30000000000000004
```

### 3. `boolean`

Solo dos valores:
```js
true
false
```

### 4. `undefined`

Significa: la variable existe pero no tiene valor asignado.

```js
let x;
console.log(x);  // undefined
```

### 5. `null`

Significa: el valor es intencionalmente vacío.

```js
let usuario = null;
```

**Diferencia:**
| Tipo | Significado |
|------|-------------|
| `undefined` | falta de valor por defecto |
| `null` | falta de valor intencional |

### 6. `symbol` (ES6+)

Crea identificadores únicos.

```js
let id = Symbol("id");
```

Incluso dos símbolos iguales no se igualan:
```js
Symbol("a") === Symbol("a");  // false
```

### 7. `bigint`

Para números enormes que superan el límite de `number`.

```js
let big = 123456789012345678901234567890n;
```

---

## Características de los Primitivos

### 1. Inmutabilidad

Los valores no cambian, se reemplazan:

```js
let a = 10;
a = 20;  // a no cambia, se crea un nuevo valor
```

### 2. Paso por valor

```js
let x = 5;
let y = x;
y = 10;

console.log(x);  // 5
```

Crear una copia exacta → no comparten memoria.

---

## Resumen

| Tipo | Comportamiento |
|------|----------------|
| Primitivo | Inmutable, copia real |
| Objeto | Mutable, copia por referencia |
