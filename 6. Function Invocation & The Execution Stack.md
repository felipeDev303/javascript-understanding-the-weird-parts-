# Function Invocation & The Execution Stack

## Function Invocation (Invocación de función)

**Definición:**  
Ejecutar una función explícitamente mediante una llamada.

**Ejemplo:**
```js
miFuncion();
```

### Proceso interno al invocar una función

Cuando JavaScript encuentra una llamada como `sumar(2, 3)`:

1. Se crea un nuevo **Execution Context** para esa función
2. Este contexto incluye:
   - Variable Environment (variables locales)
   - Lexical Environment (scope)
   - this binding
3. El Execution Context se coloca encima de la **Execution Stack**
4. JavaScript ejecuta el código de la función
5. Al terminar, el contexto se elimina de la pila

---

## The Execution Stack (Call Stack)

**Definición:**  
Estructura interna donde JavaScript organiza qué parte del programa está ejecutando en cada momento.

**Tipo de estructura:**  
Pila (Stack) con comportamiento LIFO (Last In, First Out)

### Funcionamiento

1. Primero se crea el **Global Execution Context** (GEC)
2. Cuando llamas a una función, se crea un nuevo contexto y se apila arriba
3. Si esa función llama a otra, se crea otro contexto encima
4. Cuando una función termina, su contexto se desapila
5. Cuando la pila vuelve al global, el programa está en reposo

### Ejemplo

```js
function a() {
  b();
}

function b() {
  console.log("Hola");
}

a();
```

**Execution Stack paso a paso:**

| Paso | Stack (arriba = ejecutando ahora) |
|------|-----------------------------------|
| 1 | GEC |
| 2 | GEC → a() |
| 3 | GEC → a() → b() |
| 4 | (b termina) → GEC → a() |
| 5 | (a termina) → GEC |
| 6 | (fin del programa) |

---

## Resumen

| Concepto | Descripción |
|----------|-------------|
| **Function Invocation** | Ejecutar una función creando un Execution Context |
| **Execution Stack** | Pila LIFO que organiza los Execution Contexts |
