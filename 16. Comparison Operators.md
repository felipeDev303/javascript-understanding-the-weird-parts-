# Comparison Operators

## Definición

Operadores que comparan dos valores y devuelven un resultado booleano (`true` o `false`).

---

## Tipos de comparación

### 1. Comparación de Igualdad "Floja" (==, !=)

Compara valores después de realizar coerción de tipos.

```js
5 == "5"; // true (string "5" se convierte a number)
true == 1; // true
false == 0; // true
null == undefined; // true
```

**Problema:** Puede generar resultados inesperados.

---

### 2. Comparación de Igualdad Estricta (===, !==)

Compara valor y tipo sin coerción.

```js
5 === "5"; // false (diferentes tipos)
true === 1; // false
false === 0; // false
null === undefined; // false
```

**Recomendación:** Usar siempre `===` y `!==`.

---

### 3. Comparación Relacional (<, >, <=, >=)

Comparan valores numéricos u orden alfabético.

```js
5 > 3; // true
"a" < "b"; // true
10 <= 10; // true
```

**Con diferentes tipos:**

```js
"10" > 5; // true (string se convierte a number)
```

---

## Casos especiales

### Comparación con `NaN`

```js
NaN == NaN; // false
NaN === NaN; // false
```

Para verificar NaN, usar `Number.isNaN()`:

```js
Number.isNaN(NaN); // true
```

### Comparación con `null` y `undefined`

```js
null == undefined; // true
null === undefined; // false

null == 0; // false
undefined == 0; // false
```

### Comparación de objetos

Los objetos se comparan por referencia, no por contenido:

```js
{} == {}            // false (diferentes referencias)
[] == []            // false

let obj1 = {};
let obj2 = obj1;
obj1 == obj2        // true (misma referencia)
```

---

## Tabla resumen

| Operador | Descripción          | Coerción |
| -------- | -------------------- | -------- |
| `==`     | Igualdad floja       | Sí       |
| `!=`     | Desigualdad floja    | Sí       |
| `===`    | Igualdad estricta    | No       |
| `!==`    | Desigualdad estricta | No       |
| `<`      | Menor que            | Sí       |
| `>`      | Mayor que            | Sí       |
| `<=`     | Menor o igual que    | Sí       |
| `>=`     | Mayor o igual que    | Sí       |

---

## Mejores prácticas

1. Usar siempre `===` y `!==` en lugar de `==` y `!=`
2. Tener cuidado con comparaciones de diferentes tipos
3. Usar `Number.isNaN()` para verificar NaN
4. Recordar que objetos se comparan por referencia
