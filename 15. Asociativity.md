# Operator Precedence & Associativity

## Operator Precedence (Precedencia)

**Definición:**  
Reglas que determinan qué operador se evalúa primero cuando hay varios en una expresión.

**Ejemplo:**
```js
2 + 3 * 4
```

El `*` tiene mayor precedencia que `+`, por lo tanto:
```
3 * 4 → 12
2 + 12 → 14
```

### Tabla de precedencia (de mayor a menor)

| Nivel | Operador | Ejemplos |
|-------|----------|----------|
| 19 | `()` | agrupar expresiones |
| 18 | `.` `[]` `new` | acceso a propiedades, arrays |
| 17 | `()` | llamada a función |
| 15 | `!` `+` `-` `typeof` | operadores unarios |
| 14 | `**` | exponentes |
| 13 | `*` `/` `%` | multiplicación, división, resto |
| 12 | `+` `-` | suma, resta |
| 10 | `<` `<=` `>` `>=` | comparaciones |
| 9 | `==` `!=` `===` `!==` | igualdad |
| 8 | `&&` | AND lógico |
| 7 | `\|\|` | OR lógico |
| 4 | `=` `+=` `-=` | asignación |
| 3 | `? :` | ternario |

**Regla:** Cuando dudes, usa paréntesis.

## Associativity (Asociatividad)

**Definición:**  
Dirección en que se procesan operadores con la misma precedencia.

### Left-to-Right (izquierda a derecha)

Mayoría de operadores binarios: `+ - * / % == !== && || < > <= >=`

**Ejemplo:**
```js
10 - 5 - 2
```

Se evalúa:
```
(10 - 5) - 2 → 5 - 2 → 3
```

### Right-to-Left (derecha a izquierda)

Operadores: `= += -= ! typeof **`

**Ejemplo con asignación:**
```js
a = b = c = 10;
```

Se evalúa:
```
c = 10
b = 10
a = 10
```

**Ejemplo con exponentes:**
```js
2 ** 3 ** 2
```

Se evalúa:
```
3 ** 2 → 9
2 ** 9 → 512
```

---

## Resumen

- **Precedence:** qué operador se evalúa primero
- **Associativity:** en qué dirección se evalúan operadores con la misma precedencia
- `*` y `/` se ejecutan antes que `+` y `-`
- La asignación (`=`) se evalúa de derecha a izquierda
- El operador `**` también es right-to-left
- Usar paréntesis evita errores
